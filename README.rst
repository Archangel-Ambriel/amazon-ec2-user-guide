    <article class="markdown-body entry-content p-3 p-md-6" itemprop="text"><a name="user-content-python-enhancement-proposals"></a>
<h2><a id="user-content-python-enhancement-proposals" class="anchor" aria-hidden="true" href="#python-enhancement-proposals"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Python Enhancement Proposals</h2>
<a href="https://travis-ci.org/python/peps" rel="nofollow"><img alt="https://travis-ci.org/python/peps.svg?branch=master" src="https://camo.githubusercontent.com/0f607f42b058a8ddd96c0feee2729e4270aab3b2/68747470733a2f2f7472617669732d63692e6f72672f707974686f6e2f706570732e7376673f6272616e63683d6d6173746572" data-canonical-src="https://travis-ci.org/python/peps.svg?branch=master" style="max-width:100%;"></a>
<p>The PEPs in this repo are published automatically on the web at
<a href="http://www.python.org/dev/peps/" rel="nofollow">http://www.python.org/dev/peps/</a>.  To learn more about the purpose of
PEPs and how to go about writing a PEP, please start reading at PEP 1
(<code>pep-0001.txt</code> in this repo).  Note that PEP 0, the index PEP, is
now automatically generated, and not committed to the repo.</p>
<a name="user-content-contributing-to-peps"></a>
<h2><a id="user-content-contributing-to-peps" class="anchor" aria-hidden="true" href="#contributing-to-peps"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Contributing to PEPs</h2>
<p>See the <a href="/python/peps/blob/master/CONTRIBUTING.rst">Contributing Guidelines</a>.</p>
<a name="user-content-restructuredtext-for-peps"></a>
<h2><a id="user-content-restructuredtext-for-peps" class="anchor" aria-hidden="true" href="#restructuredtext-for-peps"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>reStructuredText for PEPs</h2>
<p>Original PEP source should be written in reStructuredText format,
which is a constrained version of plaintext, and is described in
PEP 12.  Older PEPs were often written in a more mildly restricted
plaintext format, as described in PEP 9.  The <code>pep2html.py</code>
processing and installation script knows how to produce the HTML
for either PEP format.</p>
<p>For processing reStructuredText format PEPs, you need the docutils
package, which is available from <a href="http://pypi.python.org" rel="nofollow">PyPI</a>.
If you have pip, <code>pip install docutils</code> should install it.</p>
<a name="user-content-generating-the-pep-index"></a>
<h2><a id="user-content-generating-the-pep-index" class="anchor" aria-hidden="true" href="#generating-the-pep-index"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Generating the PEP Index</h2>
<p>PEP 0 is automatically generated based on the metadata headers in other
PEPs. The script handling this is <code>genpepindex.py</code>, with supporting
libraries in the <code>pep0</code> directory.</p>
<a name="user-content-checking-pep-formatting-and-rendering"></a>
<h2><a id="user-content-checking-pep-formatting-and-rendering" class="anchor" aria-hidden="true" href="#checking-pep-formatting-and-rendering"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Checking PEP formatting and rendering</h2>
<p>Do not commit changes with bad formatting.  To check the formatting of
a PEP, use the Makefile.  In particular, to generate HTML for PEP 999,
your source code should be in <code>pep-0999.rst</code> and the HTML will be
generated to <code>pep-0999.html</code> by the command <code>make pep-0999.html</code>.
The default Make target generates HTML for all PEPs.</p>
